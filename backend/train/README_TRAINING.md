# LoRA 微调说明（第二阶段再做）

这部分不是你现在第一步必须做的内容。

第一步永远是：

1. 先把聊天软件跑起来
2. 先把角色系统跑顺
3. 先整理角色样例对话
4. 最后再训练 LoRA

---

## 这个目录里有什么

- `sample_roleplay_dataset.jsonl`
  - 示例训练数据
- `export_chat_for_training.py`
  - 把聊天记录导出成训练数据的一个基础脚本
- `train_lora_unsloth.py`
  - 使用 Unsloth 的 LoRA 训练模板

---

## 建议的实际顺序

### 第一步：手工准备高质量样例对话

你最需要的不是“大量垃圾数据”，而是“高质量角色对话”。

建议每个角色先准备这些类型：

- 初见
- 日常聊天
- 安慰
- 生气
- 吃醋
- 害羞
- 邀请出门
- 节日问候
- 好感上升后的聊天

---

### 第二步：整理成 JSONL

每一行一个样本，格式参考：

```json
{"messages": [{"role": "system", "content": "..."}, {"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}]}
```

---

### 第三步：准备 GPU 环境

LoRA 微调一般建议：

- NVIDIA 显卡
- 足够显存
- Linux / WSL / 服务器都可以

---

### 第四步：修改训练脚本里的模型名和路径

把：

- `BASE_MODEL`
- `DATASET_PATH`
- `OUTPUT_DIR`

改成你自己的。

---

### 第五步：训练后导出 Adapter

训练完成后你会得到一个 LoRA Adapter 文件夹。

后续你可以：

- 用 vLLM / Transformers 推理
- 或者转换后接你自己的推理服务

---

## 重要提醒

你的角色像不像，不只是看 LoRA。

更重要的是：

- 角色卡是否写得清楚
- 样例对话是否足够稳定
- 记忆系统是否正确
- 情绪状态机是否合理

所以不要把全部希望压在微调上。
